# Repository Instructions

- All source code, code comments, public API documentation comments, CLI output, validation issue titles/details, error messages, test names, and test assertions must be written in English.
- Do not add localized user-facing strings to this SDK. If a downstream application needs localization, keep translations outside this repository and map stable issue codes there.
- When adding or changing supported EPC media formats, update the shared core rules first, then update validator diagnostics and tests for manifest paths, MIME types, content files, hash entries, and archive validation.
